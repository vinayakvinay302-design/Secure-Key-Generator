# Secure Key Generator A zero-dependency, zero-network web tool for generating cryptographically secure API keys, tokens, and secrets entirely in the browser. ## Why this exists Most "online key generator" tools send your request to a server. This one doesn't. Every value is produced locally using the browser's Web Crypto API (`crypto.getRandomValues()` / `crypto.randomUUID()`) — the same CSPRNG primitive used by TLS and WebAuthn. Nothing is transmitted, logged, written to `localStorage`, or persisted anywhere. Close the tab and the output is gone. ## Features - Six formats — hex, base64url, alphanumeric, numeric, UUID v4, or a fully custom character set - Adjustable length (8–128 characters) and optional prefix (e.g. `sk_live_`) - Batch generation — up to 10 keys at once, with copy-individual or copy-all - Live entropy meter showing bits of randomness and a strength verdict - No build step — a single static HTML file ## Security model - Randomness source: `crypto.getRandomValues()` (CSPRNG), with `crypto.randomUUID()` for UUID v4 - Network calls: none — works offline after first load - Storage: none — no cookies, no localStorage, no sessionStorage - Server component: none — this is a static file **What this tool is for:** generating dev/test API keys, session secrets, `NEXTAUTH_SECRET`/`AUTH_SECRET` values, webhook signing secrets, or any token destined for an environment variable or secrets manager. **What it isn't:** a place to store secrets. Generate here, then put the value into an environment variable, a secrets manager, or your CI/CD's encrypted secrets — never commit it to source control.
